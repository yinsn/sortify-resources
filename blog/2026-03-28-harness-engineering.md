# Harness Engineering 的底层逻辑：把世界观与价值观编码进系统

前阵子读了 OpenAI 的工程博客《[Harness Engineering: Leveraging Codex in an Agent-First World](https://openai.com/index/harness-engineering/)》。文章的核心观点很透彻：决定 Agent 系统上限的，其实已经不再是模型有多聪明，而是你如何为它设计周边的执行环境、反馈回路和边界条件。他们管这叫 Harness Engineering。

读这篇文章的时候，我正深度参与着一个名为 [Sortify](https://github.com/yinsn/sortify-resources) 的项目——一个用于推荐系统排序优化的自动化智能体。有意思的是，Sortify 的演进过程仿佛是 OpenAI 这篇文章的一个极其具体的注脚。只不过，我并不是拿着 OpenAI 的锤子去砸排序优化的钉子，而是在被复杂的业务现实毒打之后，顺着**贝叶斯决策论的哲学线索**，摸索出了极其相似的工程直觉。

这种殊途同归的感觉，挺奇妙的。我想借着这篇博客，聊聊我在 Sortify 里遇到的一些具体挣扎，以及我是如何依靠贝叶斯哲学中关于“信念”与“偏好”的思考，来构建这个“Harness”的。

### 拒绝微操：给 Agent 一个清晰的世界观

在做 Sortify 的早期，我走过弯路。推荐系统的排序优化是一个典型的非凸、多目标约束问题。一开始，我总想直接在 Prompt 里教大模型怎么去微调那些错综复杂的底层参数。结果可想而知：在混沌的高维空间里，Agent 就像个蒙着眼睛乱撞的无头苍蝇。

OpenAI 在博客里有一句话特别准：“Agent 在上下文中无法访问的东西，实际上就是不存在的。”他们管这叫 Legibility（可读性）。

在排序优化的语境里，这意味着我不能指望大模型去理解“调了这个参数，全局榜单为什么会这么变”这种玄学。为了让环境变得可读，我引入了 **Influence Share（影响力份额）** 指标，把黑箱的排序干预，变成了总和为 1 的份额兑换。你想要 2% 的 GMV？拿 3% 的广告份额来换。

更进一步，我彻底放弃了让大模型直接“猜”参数。Sortify 最终演化成了一个三层架构：人类负责定目标（战略层）；LLM 和统计算法负责做框架级的模式识别（元控制层）；而底层的脏活累活——在给定约束下找最优数值，交给了最笨但也最可靠的 Optuna 搜索器（数值执行层）。

这其实就是 Harness Engineering 的第一课：**不要试图提高 Agent 的微操水平，而是去重塑它所面对的物理法则。**

### 致命的混淆：当线上指标跌了，到底是谁的错？

在构建反馈回路时，我遇到了一个更棘手的认知危机，这也是我引入哲学视角的契机。

在早期的调参系统里，如果线上 GMV 跌了，系统的本能反应往往是“收紧约束”。但这在哲学上，其实犯了一个根本的认识论错误。它把两个截然不同的东西搅在了一起：
1. **对世界状态的认知（Belief）**：“我以为这组参数能让 GMV 涨，但世界并没有按我的预期运行。”（先验预测与实际观测不符）
2. **对结果的价值偏好（Preference）**：“GMV 确实按照预期跌了一点，但我没想到业务方会这么痛，导致满地告警。”（容忍度设定失准）

如果不区分这两者，系统就会发生难以察觉的“认知腐化”：要么陷入“一厢情愿”（强行扭曲对客观世界的认知来迎合目标），要么产生“酸葡萄心理”（反正达不到目标，干脆无视约束）。

这恰恰是贝叶斯决策哲学中最核心的告诫：**事实判断（是什么）与价值判断（好不好）必须被严格剥离**。智能体必须诚实地根据新证据来更新对世界的“信念”（贝叶斯更新），而绝对不能让自身的“偏好”去污染这种客观认知。主观期望效用理论（SEU）正是这一思想的经典表达。

### 将认识论变成代码：我不是在造一台更聪明的老虎机

当我讨论在动态环境中寻找最优解时，业界最直觉的反应往往是多臂老虎机（Bandit）。在早期，我也想过：能不能让 Agent 在真实流量里把参数当成“臂”去押注？

但顺着 Harness 的思路深挖下去，我发现两者根本不在一个层级。Bandit 算法优化的是**选择**，它直接学习“动作到收益”的映射，就像一个精明的**赌场经理**，在线上盯着哪台机器今天 payout 更高。面对环境的变化，它通常只是让旧数据衰减；它的记忆里，存的是各个选项的历史回报。

而 Sortify 优化的，是**选择背后的认识论与价值结构**。它的设计更像是一个**物理学家兼宪法设计者**。

在架构上，Sortify 将贝叶斯哲学化作最底层的不变量，强制执行了事实（Belief）与价值（Preference）的物理分离，做出了一个正交的**双通道机制**：

- **信念通道（校准世界模型）**：回答“离线指标在线上到底能兑现多少”。当预测与实际产生偏差时，LLM 通过输出 `delta_intercept` 来调整映射截距。它不是在盲目试错，而是在做贝叶斯意义上的信念更新：“地图错了，我们需要重新对齐客观认知。”
- **偏好通道（校准主观效用）**：回答“约束被碰一下到底有多痛”。当业务红线被击穿时，LLM 通过 `penalty_multiplier` 放大惩罚权重。它是在调整系统的价值函数：“现实的痛感超出预期，我们的底线该更硬了。”

这种分离让 Sortify 与 Bandit 走向了完全不同的工程路径：
1. **重塑决策空间**：Sortify 先把“排序是被哪些力推动的”拆开。它不再评估“哪组参数更好”，而是问“100% 的排序决策力该如何在不同因子间分配”。把 5% 影响力从订单换到 GMV，变成了一个严格可定义的问题，而不是拍脑袋去拉另一只臂。
2. **离线探索与线上打标**：Bandit 的探索发生在线上，用真实用户的体验付学费；Sortify 则是把数千次的参数搜索放在离线，而每轮线上的 A/B 测试，仅仅是用来给“离线世界模型”打标注。
3. **继承判断力**：Sortify 跨轮记忆的不是上一组哪个参数赢了，而是 episode 的演变、迁移关系、惩罚历史和 LLM 的推理证据链。

即使把 Bandit 升级到 Contextual 或 Continuum-armed Bandit，核心依然是让系统学“怎么选”。而 Sortify 是两层学习：先问清楚推动排序的力是什么、现实的刻画偏没偏、红线有多硬，然后才在重塑后的优化框架里，让底层的搜索器去解题。这种架构级别的剥离，让 Agent 的每一次纠错都极具**可诊断性**，彻底告别了“按了葫芦起了瓢”的调参黑箱。

### 嵌套的闭环：Harness Engineering 给单兵作战带来的认知杠杆

当我把这一套复杂的架构在 Sortify 中实现，并真正让它在生产环境中跑起来的时候，我意识到 Harness Engineering 带来的，不仅仅是系统性能的提升，更是一种**开发范式的质变**。

这里面存在一个非常奇妙的“元嵌套”闭环。

在业务逻辑层，Sortify 是一个**目标域的智能体**。它基于 SEU 理论解耦信念与偏好，自行观察历史证据，自行诊断结构性断点，自行完成影响力置换。这是机器对传统算法工程师在系统性决策工作上的替代。

而在开发层，整个 Sortify 系统——包含近十万行代码、78 个 Python 模块、复杂的 Optuna 搜参引擎、基于 SQLite 的 7 表记忆系统，乃至端到端的自动化管线——是**由我一个人调度一组 AI 编程智能体从零构建的**。在整个项目里，没有一行生产代码出自我的手，也没有一段技术报告是我亲笔起草的。

我把在 Sortify 中设计的“三层架构（人类定战略 -> LLM 校准框架 -> 算法纯执行）”，原封不动地搬到了我自己的开发工作流中：
1. 我（人类）不再写代码，而是负责“定义意图”和“设计架构”；
2. AI 智能体负责“翻译意图并生成代码”；
3. 失败时，我不是去手动改 Bug，而是去诊断是我的“任务规约”错了，还是智能体的“执行”偏了，进而调整我的框架级参数——即任务规约与设计约束（就像在业务中调整截距和惩罚权重一样）。

这其实就是最高维度的 Harness Engineering。

当 AI 吸收了实现层面的所有沉重负担，构建复杂系统的壁垒便从“团队规模与协作成本”转移到了“单个人的问题分解能力与架构品味”。我只需要专注于想清楚“我要把什么世界观编码进系统里”，AI 就能帮我把这套世界观转化为可执行的代码。

这并不是简单的效率提升，而是**认知杠杆**的极大化。Harness Engineering 让“一个人指挥一支 AI 团队，完成原本需要 3-5 人团队花数月才能落地的系统级工程”成为了可能。

### 写在最后

回过头来看，不管是 OpenAI 在代码生成场景里提炼出的 Harness Engineering，还是我在推荐系统排序优化中依靠贝叶斯哲学摸索出的 Sortify 架构，本质都在回答同一个问题：在动态、复杂且充满噪声的真实世界里，我们该如何安置一个偶尔聪明、但极易脱轨的 LLM？

答案或许并不新鲜：限制它的手脚，结构化它的眼睛，拆分它的思考，然后把它镶嵌在一个拥有坚实哲学根基（比如贝叶斯决策论）的工程闭环里。

更有趣的是，当我用这套思路去“驾驭”开发过程中的 AI 编程助手时，它赋予了我前所未有的单兵作战能力。当人类不再被困于实现的泥沼，而成为纯粹的思想者与架构师时，这或许才是 AI 时代真正让人着迷的地方。
